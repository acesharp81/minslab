01

구독 신청은 카카오 동의가 먼저입니다

메시지를 실제로 받으려면 카카오 로그인 후 [선택] 카카오 메시지 전송 권한에 동의해야 합니다.

진행 순서

  1. 상단 메뉴에서 구독 및 케이스 신청을 엽니다.
  2. 카카오 로그인/메시지 전송 동의하기를 누릅니다.
  3. 카카오 화면에서 [선택] 카카오 메시지 전송을 선택합니다.
  4. 화면으로 돌아와 이름과 기관을 선택합니다.
  5. 구독 대상 케이스를 전체 기준으로 다시 체크하고 요청합니다.

구독 변경은 추가·삭제분만 보내는 방식이 아닙니다. 변경 후 받고 싶은 전체 케이스를 다시 체크해 주세요. 모든 체크를 해제하면 해지 요청입니다.

02

새 모니터링 주제는 케이스 신청 게시판에 제안합니다

일반 사용자는 닉네임으로 필요한 케이스를 제안하고, 관리자는 이를 바로 새 케이스 등록창으로 가져갈 수 있습니다.

사용자 입력

닉네임, 수정·삭제 암호, 케이스 제목, 케이스 프롬프트, 키워드, 필수키워드를 입력합니다. 암호는 평문 저장하지 않습니다.

목록 표시

목록에는 제목과 닉네임만 먼저 보이고, 글을 클릭하면 프롬프트와 키워드가 펼쳐집니다.

클린 AI

등록·수정 글은 로컬 LLM이 유휴 시간에 검사합니다. 문제 표현이 있으면 “클린 AI에 의해서 블라인드 처리되었습니다.”로 잠깁니다.

관리자 등록

관리자 로그인 상태에서는 케이스 등록 버튼으로 관리 설정의 새 케이스 창을 열고 신청 내용을 자동 입력할 수 있습니다.

03

대시보드는 오늘 처리 상태와 기사별 결과를 봅니다

공통분석, 임베딩, 케이스 판정, 발송 단계가 어디까지 진행됐는지 한눈에 확인합니다.

중요한 표시

  • + 케이스: 케이스별 후보 제외, 발송 제외, 발송 완료 상태를 봅니다.
  • 유사 기사 묶어서 보기: 기본 선택이며, 임베딩 완료 후 의미 유사도 기준으로 갱신됩니다.
  • 재분석: 관리자만 사용할 수 있으며 기사 단위로 모든 케이스 판정을 다시 대기 상태로 돌립니다.
  • 관련 보도자료: 보도자료 유사도 기준을 넘은 공식 자료를 연결합니다.
04

신경망 분석과 보도동향은 관계를 보는 화면입니다

대시보드가 개별 기사 중심이라면, 신경망과 보도동향은 기사 묶음과 공식 보도자료의 관계를 보여줍니다.

신경망 분석

기관, 케이스, 기간을 선택하면 발송 완료 기사 중심의 3D 의미 관계망을 봅니다. 관련 기사끼리만 묶이도록 유사도 기준을 적용합니다.

주요 화두

노드와 화두를 함께 보며 해당 케이스에서 반복되는 주제와 단어를 확인합니다.

보도동향 검색

제목, 담당과, 담당자 이름으로 행안부 보도자료를 검색할 수 있습니다.

관련 기사

보도자료별 관련 기사 수와 Markdown 원문을 확인해 공식 발표와 언론 보도의 연결을 봅니다.

05

관리자는 회원, 케이스, 모델, 공지를 한 곳에서 관리합니다

관리 설정은 기존 홈페이지 관리자 로그인 세션으로 보호됩니다.

관리자 업무

  • 회원 이름은 기본 마스킹, 이름 표시 체크 시 전체 표시
  • 회원 행 클릭 후 케이스별 구독 체크박스 조정
  • 기관 아래 케이스를 드래그해 표시 순서 변경
  • 공통분석·케이스판정·예비모델·임베딩 모델 설정
  • 공지사항 등록, 내리기, 삭제
  • 관련 보도자료 기준과 유사 기사 묶음 기준 조정
06

자주 묻는 질문

운영 중 헷갈리기 쉬운 지점을 짧게 정리했습니다.

발송 건수는 기사 수인가요?

아닙니다. 실제 수신자에게 발송되는 메시지 단위입니다. 수신자가 3명이면 기사 1건도 발송 3건이 될 수 있습니다.

LLM 0%는 항상 무관하다는 뜻인가요?

아닙니다. 판정 실패나 중단 상태일 수 있으므로 케이스 상태가 completed인지 먼저 확인해야 합니다.

관리 설정이 늦게 뜨면 어떻게 하나요?

현재는 저장값을 먼저 보여주고 모델 상태는 순차 확인합니다. 화면은 먼저 열리고, 모델 상태 문구가 하나씩 갱신됩니다.

토큰이 다 떨어지면 어떻게 되나요?

주 모델이 소진되면 예비 모델로 전환합니다. 모든 제공자가 소진되면 영업중지 메시지가 표시되고 다음 초기화 시각 이후 재개합니다.
